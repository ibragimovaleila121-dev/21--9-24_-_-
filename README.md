## 1. Описание проекта
Данный проект представляет собой прототип информационной системы (ИС) для автоматизации учета товаров, складских остатков и продаж в магазине электроники. 
Система позволяет структурировать хранение данных о товарах, брендах, остатках на складе и истории продаж, а также быстро получать аналитическую информацию через SQL-запросы.

**Основные возможности:**
- Просмотр каталога товаров с ценами и остатками
- Поиск товаров по названию или категории
- Оформление продаж (автоматическое списание со склада)
- Управление товарами, брендами и складскими остатками
- Формирование отчетов по продажам и остаткам

## 2. Техническое задание (ТЗ) вкратце
*   **Цель:** Перейти от бумажного учета к автоматизированной базе данных на SQLite с удобным консольным интерфейсом.
*   **Основные функции:**
    1. Регистрация новых записей (товары, бренды, продажи).
    2. Поиск данных по названию или категории товара.
    3. Формирование итоговых отчетов (выручка по брендам, остатки на складе).
*   **Требования к БД:** Минимум 4 связанные таблицы, соблюдение целостности данных (FOREIGN KEY, CHECK-ограничения).

## 3. Структура базы данных (ER-схема)
Проект базируется на следующих сущностях:
1.  **brands (Бренды)** — хранит данные о производителях техники (название, страна, год основания).
2.  **products (Товары)** — содержит информацию о товарах (название, цена, категория).
3.  **inventory (Склад)** — отвечает за учет остатков товаров на складе.
4.  **sales (Продажи)** — фиксирует историю продаж (какой товар, сколько, кому, на какую сумму).

**Связи между таблицами:**
- `brands` → `products`: один бренд может производить много товаров (1:N)
- `products` → `inventory`: у одного товара может быть только один остаток (1:1)
- `products` → `sales`: один товар может участвовать во многих продажах (1:N)

*Скриншот ER-диаграммы находится в папке `/diagrams`.*

## 4. Реализованные SQL-запросы
В рамках проекта подготовлены сценарии для:
*   **JOIN (объединение таблиц):** Вывод товаров с названиями брендов и остатками на складе.
*   **GROUP BY (группировка):** Подсчет общей выручки по каждому бренду.
*   **Подзапрос:** Выборка товаров дороже средней цены.
*   **Фильтрация данных по условию:** Товары с остатком меньше 10 штук (нуждаются в закупке).
*   **TOP-N запрос:** Вывод 3 самых продаваемых товаров.

### Примеры сложных запросов:

```sql
-- Выручка по брендам (JOIN + GROUP BY)
SELECT b.name, SUM(s.total_amount) as total_revenue
FROM sales s
JOIN products p ON s.product_id = p.product_id
JOIN brands b ON p.brand_id = b.brand_id
GROUP BY b.name
ORDER BY total_revenue DESC;

-- Товары дороже среднего (подзапрос)
SELECT name, price
FROM products
WHERE price > (SELECT AVG(price) FROM products);

-- Самые продаваемые товары (TOP-3)
SELECT p.name, SUM(s.quantity_sold) as total_sold
FROM sales s
JOIN products p ON s.product_id = p.product_id
GROUP BY p.name
ORDER BY total_sold DESC
LIMIT 3;
