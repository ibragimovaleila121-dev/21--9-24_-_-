## 1. Описание проекта
Данный проект представляет собой прототип информационной системы (ИС) для автоматизации учета товаров, складских операций и продаж в магазине электроники.
Система позволяет хранить информацию о товарах, брендах, складах и продажах, а также быстро получать данные о наличии товаров и выручке с помощью SQL-запросов.
##  2. Техническое задание (ТЗ) вкратце
*  Цель: Перейти от ручного учета товаров и продаж к автоматизированной базе данных на SQLite.
Основные функции:
Регистрация новых товаров, брендов и продаж.
Поиск товаров по названию, бренду или наличию на складе.
Формирование отчетов по продажам и остаткам товаров.
Требования к БД:
Минимум 4 связанные таблицы.
Связи между таблицами через внешние ключи.
Обеспечение целостности данных.

##  3. Структура базы данных (ER-схема)
Проект базируется на следующих сущностях:

Товары — хранит информацию о товарах (название, цена, бренд, склад).
Бренды — содержит данные о производителях товаров.
Склады — хранит информацию о местах хранения товаров.
Продажи — фиксирует факты продажи товаров (дата, количество, сумма).

Связи:

Товары связаны с брендами (каждый товар имеет бренд).
Товары связаны со складами (товар хранится на складе).
Продажи связаны с товарами (каждая продажа относится к товару).

*Скриншот ER-диаграммы находится в папке `/diagrams`.*

##  4. Реализованные SQL-запросы
В рамках проекта подготовлены сценарии для:
*   Подсчета общего количества [товаров/услуг].
*   Выборки данных с использованием `JOIN` (объединение таблиц).
*   Фильтрации данных по условию (например, цена > 1000).

##  Структура репозитория
*   `/data` — файл базы данных `project.db`.
*   `/sql` — скрипты создания таблиц (`create.sql`) и тестовые данные (`insert.sql`).
*   `/diagrams` — графическая схема базы данных.
*   `queries.sql` — файл с примерами сложных запросов.

##  Как запустить (Инструкция)
1. Установите **DB Browser for SQLite**.
2. Склонируйте репозиторий: `git clone [ссылка на ваш репозиторий]`.
3. Откройте файл из папки `/data` и выполните запросы из файла `queries.sql`.
